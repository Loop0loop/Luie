# Memory Eval Writer Pain Point Taxonomy

이 문서는 RAG + Memory Engine의 Phase 1-1 기준 taxonomy다.

목적은 웹소설 작가가 실제 집필 중 자주 겪는 설정 사고를 eval case로 고정하는 것이다. 각 category는 `WRITER_PAIN_POINT_TAXONOMY`의 `key`와 동일해야 한다.

## Seed 기준

- 각 category는 최소 10개 seed case를 만들 수 있어야 한다.
- 모든 seed case는 gold evidence를 가져야 한다.
- case name에는 category key가 안정적으로 들어가야 한다.
- 질문은 기술 문장이 아니라 작가가 실제로 물을 법한 문장이어야 한다.

## Categories

### alias-confusion

작가 문제: 같은 인물의 본명, 별명, 직함, 호칭을 서로 다른 인물처럼 쓰거나 반대로 다른 인물을 같은 인물처럼 쓰는 문제.

작가 질문 예시:

```text
이 장면의 폐하와 카이렌이 같은 사람이라고 봐도 돼?
여기서 선배라고 부르는 사람이 앞 장면의 류하 맞아?
```

### knowledge-state

작가 문제: 캐릭터가 아직 모르는 정보를 알고 있는 것처럼 쓰는 문제.

작가 질문 예시:

```text
12화 기준으로 여주가 약의 정체를 알고 있다고 써도 돼?
이 시점에서 주인공이 흑막 이름을 알 근거가 있어?
```

### future-leakage

작가 문제: 뒤 회차에서 드러나는 정보를 앞 회차 시점 답변이나 집필 판단에 섞는 문제.

작가 질문 예시:

```text
5화 기준으로 이 조직이 배신자라는 걸 말해도 돼?
이 대답에 18화 정보를 섞으면 안 되는데 현재 근거만으로 가능해?
```

### draft-contamination

작가 문제: 초안, 폐기 설정, 작가 메모를 실제 정사 원문처럼 섞는 문제.

작가 질문 예시:

```text
이 설정 정사에 실제로 나왔어, 아니면 폐기 초안에만 있었어?
이 인물의 옛 이름을 본편에서 쓴 적 있어?
```

### relation-direction

작가 문제: 누가 누구에게 감정, 행동, 적대, 호의를 갖는지 방향을 반대로 쓰는 문제.

작가 질문 예시:

```text
A가 B를 배신한 거야, B가 A를 배신한 거야?
이 장면에서 누가 먼저 적대 행동을 했어?
```

### unresolved-thread

작가 문제: 아직 회수하지 않은 복선을 이미 해결된 사실처럼 쓰거나 회수된 떡밥을 미회수로 착각하는 문제.

작가 질문 예시:

```text
이 반지 떡밥 아직 안 풀린 거 맞아?
아버지의 편지는 이미 회수된 복선이야?
```

### continuity-state

작가 문제: 인물의 생존, 부상, 위치, 소속, 능력, 소유물이 이전 원문 상태와 충돌하는 문제.

작가 질문 예시:

```text
이 시점에 이 인물이 아직 수도에 있어야 해?
주인공이 이 검을 아직 가지고 있는 게 맞아?
```

### motivation-drift

작가 문제: 인물의 감정선이나 행동 동기가 앞선 장면의 축적 없이 갑자기 바뀌는 문제.

작가 질문 예시:

```text
여주가 갑자기 주인공을 믿는 근거가 앞에 쌓였어?
이 인물이 복수를 포기할 만한 장면이 있었나?
```

### world-rule-conflict

작가 문제: 마법, 능력, 조직 규칙, 금기, 계약 조건 같은 세계관 규칙을 장면마다 다르게 쓰는 문제.

작가 질문 예시:

```text
이 능력은 원래 밤에만 쓸 수 있는 설정 아니었어?
계약자는 거짓말을 못 한다는 규칙이 이 장면과 충돌하지 않아?
```
